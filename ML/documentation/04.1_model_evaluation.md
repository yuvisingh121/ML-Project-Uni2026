### Task 4 – Model Evaluation
Valutazione del classificatore Naive Bayes sul dataset esteso

## 0. Obiettivo del Task
In questo task viene valutato il classificatore Naive Bayes progettato manualmente nel Task 2, applicandolo al dataset esteso `training.csv` composto da 50.000 osservazioni.
Lo scopo è verificare la capacità del modello di generalizzare quando viene utilizzato su un campione molto più ampio rispetto al dataset manuale usato per stimare le probabilità.

## 1. Preparazione del dataset
Per garantire coerenza con il modello costruito nel Task 2, vengono applicate al dataset `training.csv` le stesse trasformazioni:
- discretizzazione della variabile BMI nelle categorie Normal, Overweight, Obese
- discretizzazione della variabile Age nelle categorie Bambino, Giovane, Adulto, Anziano

Queste trasformazioni permettono di utilizzare correttamente le probabilità condizionate calcolate manualmente.

## 2. Applicazione del classificatore
Per ogni osservazione vengono calcolati due punteggi:
- Score_0(Y=0) = P(Y=0) * Π P(X_i | Y=0)  
- Score_1(Y=1) = P(Y=1) * Π P(X_i | Y=1)

La classe assegnata è quella con punteggio maggiore.

Il risultato della predizione viene salvato nella colonna Predicted.

## 3. Tabella dei risultati
Per ogni riga del dataset vengono confrontati:
- valore reale: Diabetes_binary
- valore predetto: Predicted

Questa tabella costituisce la base per la costruzione della confusion matrix e delle metriche di valutazione.

## 4. Confusion Matrix
La matrice di confusione riassume il comportamento del modello distinguendo:
- True Negative (TN): non diabetici correttamente classificati
- False Positive (FP): non diabetici classificati come diabetici
- False Negative (FN): diabetici non rilevati
- True Positive (TP): diabetici correttamente classificati

La distribuzione di questi valori permette di comprendere quali errori il modello commette più frequentemente.

## 5. Metriche di valutazione
A partire dalla confusion matrix vengono calcolate le metriche standard:
- Accuracy = (TP + TN) / (TP + TN + FP + FN)
- Precision = TP / (TP + FP)
- Recall = TP / (TP + FN)
- F1-Score = 2 * (Precision * Recall) / (Precision + Recall)

Queste metriche permettono di valutare sia la qualità generale del modello sia la sua capacità di individuare correttamente i casi positivi.

## 6. Interpretazione dei risultati
Il modello Naive Bayes costruito manualmente mostra limiti evidenti quando applicato a un dataset molto più grande:
- le probabilità condizionate stimate su un campione ridotto non rappresentano adeguatamente la distribuzione reale
- la discretizzazione manuale riduce l’informazione disponibile
- alcune feature risultano molto più informative nel dataset completo rispetto al dataset manuale
Questi fattori spiegano eventuali squilibri tra falsi positivi e falsi negativi e una performance complessiva non ottimale.

## 7.Conclusioni
Il classificatore Naive Bayes manuale costituisce un valido esercizio didattico per comprendere il funzionamento del modello, ma non è competitivo quando applicato a un dataset reale di grandi dimensioni.
Il passo successivo consiste nel confrontare queste prestazioni con modelli implementati tramite librerie standard come Scikit-Learn.
